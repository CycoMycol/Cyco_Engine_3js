The Unreal Engine Volumetric Cloud and Environment system is built as a physically-based, material-driven architecture that unifies atmospheric scattering, cloud volumes, and dynamic lighting. Instead of relying on traditional 2D skyboxes, it relies on a screen-space rendering pipeline that uses 3D ray-marching through volume textures to handle transitions from ground level all the way to outer space.🏛️ Core Architecture & Component CohesionThe environment system is designed around four tightly coupled components that pass data to each other in real time via the Environment Light Mixer:Sky Atmosphere: Simulates physically accurate planetary atmospheres using Rayleigh and Mie scattering models to produce realistic time-of-day sky colors.Directional Light: Serves as the primary celestial body (Sun or Moon), sending light vector data directly into the atmosphere and cloud layers.Volumetric Cloud: A specialized engine component that generates a screen-space proxy geometry onto which ray-marched mathematical shapes are rendered.Sky Light: Captures the completed sky and cloud output in real time using Real Time Capture Mode, ensuring accurate ambient bounced lighting reflects across the entire scene.☁️ How the Volumetric Cloud System Was BuiltThe technical architecture of the Volumetric Cloud Component balances visual flexibility with rigid runtime optimizations:1. Screen-Space Proxy GeometryThe system does not render millions of voxels globally.It generates a single screen-aligned triangle scaled to cover the player's view.The clouds are rendered as a post-reflection/translucency effect directly on this screen space canvas.2. Ray-Marching & 3D Texture SamplingInside the designated atmosphere layer (defined by an inner and outer planetary radius), the engine casts rays from the camera.As each ray steps through the cloud layer, it samples a specialized Volume Texture (\(3\text{D Texture}\)) containing procedural noise.The default look is driven by m_SimpleVolumetricCloud, located within the engine's core system folders.3. Noise-Driven ModelingBase Shape: Uses low-frequency Perlin or Worley noise to establish large-scale cloud formations and coverage masks.Erosion & Detail: Applies high-frequency Worley noise at the edges to simulate wind erosion, whispy vapor tails, and fine billows.4. Physically-Based Lighting CalculationsPhase Functions: Employs the Henyey-Greenstein phase function to calculate how light scatters forward or backward when colliding with moisture droplets.Shadow Tracing: For every step along the primary camera ray, secondary rays are cast toward the Directional Light to compute light absorption, giving clouds deep, realistic self-shadowing and dark bellies.🌫️ Volumetric Fog vs. Volumetric LightmapsUnreal Engine splits regional environment lighting into two separate pipelines depending on whether objects are static or dynamic:                  ┌──────────────────────────────────────────┐

Volumetric Fog (Real-Time GPU)Builds a camera-aligned 3D Voxel Grid structured within the camera's view frustum.Tracks dynamic point lights, spot lights, and directional lights frame-by-frame.Allows localized control via properties like Volumetric Scattering Intensity on individual light sources to punch out specific light shafts without fogging the whole scene.Volumetric Lightmaps (Precomputed CPU/GPU)Utilized for static global illumination via Lightmass.Computes an advanced 3D grid of lighting samples across the designated Lightmass Importance Volume.Places higher sample densities near geometry to ensure moving characters or dynamic objects seamlessly receive realistic indirect lighting when transitioning out of shadow structures.🛠️ Official Workflow & Implementation RulesTo manipulate or construct scenes using this subsystem, Epic Games establishes a definitive implementation strategy via the Unreal Engine Documentation:Do Not Overwrite Engine Shaders: The core cloud shader system lives under Engine Content > EngineSky > VolumetricClouds. Never edit m_SimpleVolumetricCloud directly.Use Material Instances: Create local Material Instances inside your project's local content folder to adjust cloud parameters safely.Adjust Scaling Conservatively: Define explicit constraints within the Cloud Component Details panel using parameters like Altitude Start and Altitude End (measured strictly in kilometers) to keep the ray-march loops performant and aligned to the horizon.
